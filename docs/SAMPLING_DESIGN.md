# Extended Corpus Sampling Design

**Version:** 1.0  
**Date:** 2026-06-26  
**Status:** Implemented — supersedes the arbitrary 5-target list used before DECISION-143

---

## Motivation

The initial extended corpus (`HIP17147`, `HIP39826`, `HIP66704`, `HIP74981`,
`HIP82860`) was assembled for corpus *diversity* (non-Cygnus sky regions),
not statistical *sampling*.  Without a documented sampling rationale, any
null result or candidate claim from these targets cannot be defended before
peer review.  This document specifies a stratified random sampling design
that replaces that list.

---

## Sampling Frame

**Source:** Breakthrough Listen High-Priority Candidate (HPRC) target list  
**Citation:** Isaacson et al. 2017, PASP 129, 054501  
**Seed file:** `data/bl_hprc_seed_targets.csv`  
**Coverage:** 48 targets from the HPRC list with confirmed or probable BL
GBT L-band data availability, drawn from Enriquez et al. 2017 (ApJ 849 104),
Price et al. 2020 (AJ 159 86), and direct HPRC catalog inspection.

The seed file is a static, committed artifact.  Its SHA-256 checksum is
recorded in every manifest generated by `build_stratified_sample.py`.

---

## Stratification Scheme

Targets are stratified by the Cartesian product of three dimensions:

### Dimension 1 — Stellar Distance

| Bin label | Range (pc) | Rationale |
|-----------|-----------|-----------|
| `near`    | 0 – 8     | Nearest stars; highest EIRP sensitivity; fewest known at 1.4 GHz due to small sample size |
| `mid`     | 8 – 20    | Core SETI search volume; richest in HPRC candidates |
| `far`     | 20 – 50   | Extended coverage; tests pipeline at lower SNR margin |

### Dimension 2 — Spectral Type

| Class | Description | SETI rationale |
|-------|-------------|----------------|
| F     | 6000–7500 K main sequence | Detectable habitable zones; shorter lifetimes than G/K |
| G     | 5200–6000 K (Sun-like) | Canonical SETI target; well-studied |
| K     | 3700–5200 K | Longer lifetimes than G; increasingly favoured in SETI literature |
| M     | <3700 K    | Most numerous; habitable zones close-in; included for completeness |

### Dimension 3 — Exoplanet Host Status

| Value | Meaning |
|-------|---------|
| 0     | No confirmed exoplanets (per NASA Exoplanet Archive at time of seed-file creation) |
| 1     | Confirmed exoplanet host |

This yields **24 strata** (3 distance bins × 4 spectral classes × 2 exoplanet values).

---

## Sampling Algorithm

```
for each stratum:
    1. Filter HPRC seed list to targets matching all three stratum criteria
    2. Sort by distance (nearest first) as a secondary ordering
    3. Sample min(per_stratum_quota, available_count) targets
       without replacement using Python random.Random(seed)
```

Default parameters (used for the committed manifest):

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| `--seed` | 42 | Fixed for reproducibility; reported in any publication |
| `--per-stratum` | 2 | Balances coverage vs. total download burden |
| Resulting selection | ≤48 targets | Depends on seed file coverage of each stratum |

## Generating the Manifest

```bash
# Regenerate from scratch (output: data/target_sample_manifest.json)
.venv/bin/python scripts/build_stratified_sample.py

# Inspect without writing
.venv/bin/python scripts/build_stratified_sample.py --dry-run

# Larger corpus (3 per stratum, max 60 total)
.venv/bin/python scripts/build_stratified_sample.py \
    --per-stratum 3 --max-targets 60
```

The manifest is deterministic: same seed CSV + same random seed → same output.

## Downloading the Extended Corpus

The download script reads from the manifest:

```bash
git pull origin main
caffeinate -i bash scripts/download_bl_extended_corpus.sh \
    --manifest data/target_sample_manifest.json \
    --discover-only \
    --availability-output /tmp/bl_hdf5_availability.tsv

git pull origin main
caffeinate -i bash scripts/download_bl_extended_corpus.sh \
    --manifest data/target_sample_manifest.json
```

Targets without a discoverable BL open-data HDF5 URL are skipped with a
warning.  The download script reports the fraction of manifest targets that
had available data.  When `TECHNO_EXTENDED_CORPUS_MAX_TARGETS` is set, the
limit applies to new URL-available HDF5 downloads rather than raw manifest
position or already-downloaded evidence, so unavailable manifest entries and
local reuses do not consume the bounded download budget.
The optional `/tmp/bl_hdf5_availability.tsv` file is a local, no-payload map of
verified target-to-HDF5 URLs for the current BL search-page state.

## Reporting Requirements

Any publication or candidate submission using data produced by this sampling
scheme must state:

1. Sampling frame: "BL HPRC target list (Isaacson et al. 2017)"
2. Stratification: "Stratified random sample across distance (near/mid/far),
   spectral class (FGKM), and exoplanet-host status; 2 targets per stratum"
3. Random seed: "Python `random.Random(42)`; reproducible from committed
   manifest `data/target_sample_manifest.json` (SHA-256 recorded therein)"
4. Data availability caveat: "Not all sampled targets had BL open-data HDF5
   files available at download time; Table N reports which were obtained"

---

## What This Design Does NOT Claim

- It is **not** a volume-complete survey (seed file is a subset of HPRC).
- It is **not** sensitivity-matched across distance bins (different SNR margins).
- It does **not** guarantee BL open-data availability for every drawn target.
- It does **not** constitute a detection claim or authorize external submission.

## Scientific Guardrail

All outputs of this sampling design — manifests, downloaded HDF5 files, derived
`.dat` hit tables, and candidate reports — are local scheduling aids only.  No
selected target, downloaded file, or derived hit table constitutes a
technosignature detection or authorizes external submission.  Independent review
is required before any external submission, per `docs/EXTERNAL_SUBMISSION_PROTOCOL.md`.

## Real Full-Catalog Expansion (2026-07-04)

The 48-star seed file above is a hand-picked subset of a much larger real
published target list. Isaacson et al. 2017's full stellar target table has
**1,709 real rows** (60 nearest stars + 1,649 Hipparcos stars), confirmed
via a real live VizieR query, 2026-07-04 (`docs/bl_hprc_target_list_research.md`
has the full research trail). The real, confirmed VizieR catalog identifier
is **`J/PASP/129/E4501`** (table `table1`) — a plausible-looking
`J/PASP/129/054501` identifier was tested live and confirmed *not* to exist;
do not use it.

`scripts/acquire_bl_hprc_full_catalog.py` downloads this real full table
(raw TSV preserved for provenance, plus a normalized CSV), validating both
schema and the real expected row count before writing anything. Run it with:

```bash
.venv/bin/python scripts/acquire_bl_hprc_full_catalog.py
```

**Real, honest limitation, not yet closed**: the full VizieR table provides
`Star`/`RAJ2000`/`DEJ2000`/`Ep`/`Vmag`/`SpType`/`Dist`/`pmRA`/`pmDE`/`SimbadName`
only — it does **not** include B-V color or exoplanet-host status, both of
which the current 48-star seed CSV has (supplemented from other sources for
that small hand-picked set). `scripts/build_stratified_sample.py`'s
stratification scheme currently requires both fields for its 24-cell
distance × spectral-class × exoplanet-host design. Wiring the full
1,709-row catalog into the stratified sampler therefore still needs one of:
(a) a real cross-match against a real exoplanet-host catalog (e.g. the NASA
Exoplanet Archive) to populate the exoplanet-host field for the fields, or
(b) a revised stratification scheme that drops the exoplanet-host dimension
for the expanded set. Neither has been decided yet — this is intentionally
left open rather than fabricated. No B-V/exoplanet-host value should be
invented for any of the 1,709 stars.

**Real archive policy, verified 2026-07-04, not guessed**: VizieR/CDS
publishes no documented rate limit for this table endpoint. The
Breakthrough Listen Open Data Archive itself documents a 500-file
per-query result cap and no published concurrent-request/download limit —
see `docs/bl_hprc_target_list_research.md` for the exact sources checked.
